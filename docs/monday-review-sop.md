# Monday Review SOP

This is the weekly operating procedure for the launch MVP.

## Timing

- schedule generation: Monday, 08:00, Europe/London
- send window: after editorial review and test email approval

## Review steps

1. Open the latest GitHub Actions run.
2. Read the run summary.
3. Open or download:
   - `output/issue-YYYY-MM-DD/newsletter.md`
   - `output/issue-YYYY-MM-DD/newsletter.html`
   - `output/issue-YYYY-MM-DD/review-summary.md`
4. Review:
   - issue title and date
   - logo and signature
   - editorial opening
   - article selection and order
   - British English tone
   - practical relevance for first-opinion vets
5. If needed, edit locally and regenerate the issue.
6. Paste the final HTML into a Mailchimp campaign.
7. Send test emails to:
   - the business mailbox
   - one mobile device inbox
8. If the test is clean, send manually to the audience.

## Hold criteria

Do not send if any of the following are true:

- wrong date or issue window
- off-topic papers
- broken links
- missing logo or signature
- awkward wording that would reduce trust
- rendering problems in the Mailchimp preview or test emails

## Launch standard

The final send must look:

- clinically credible
- editorial in tone
- visually consistent with Mobile Vet Referral branding
- readable on both desktop and mobile
