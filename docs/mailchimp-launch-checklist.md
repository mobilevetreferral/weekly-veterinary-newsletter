# Mailchimp Launch Checklist

Use this checklist to configure the review-first MVP in Mailchimp.

## Audience setup

Create one primary audience for the newsletter.

Recommended audience profile:

- audience name: `Mobile Vet Referral Newsletter`
- audience purpose: weekly veterinary clinical newsletter
- source tag: `website-newsletter`

Merge fields to confirm:

- `EMAIL`
- `FNAME`
- `LNAME`

## Consent and compliance

For launch, configure:

- **double opt-in** enabled
- GDPR-friendly marketing consent wording on the signup form
- unsubscribe footer enabled on every campaign
- your privacy-policy link included on the signup page

Do not launch until the full journey is tested:

1. submit form
2. receive confirmation email
3. confirm subscription
4. verify subscriber appears as subscribed
5. verify unsubscribe works from a test campaign

## Sender setup

Before the first live send:

- use the chosen existing `mobilevetreferral.co.uk` mailbox as:
  - `From`
  - `Reply-To`
- authenticate the sending domain in Mailchimp
- complete test sends to:
  - the chosen business mailbox
  - one mobile device inbox

## Signup form settings

Use an **embedded Mailchimp form** for the WordPress newsletter page.

Include fields:

- email address
- first name
- last name

Capture source consistently:

- audience tag or source field: `website-newsletter`

## Campaign workflow for v1

Each week:

1. open the latest issue generated by GitHub Actions
2. review `newsletter.md`, `newsletter.html`, and `review-summary.md`
3. create a Mailchimp campaign manually
4. paste the final HTML
5. send test emails
6. send manually only after approval

Use [mailchimp-weekly-send-playbook.md](/Users/sarahtucci/Documents/Newsletter automations/docs/mailchimp-weekly-send-playbook.md) for the exact campaign-build steps.

## Deliberate v1 exclusions

Do not configure yet:

- automatic campaign creation
- automatic send
- multiple audience segments
- API-based subscriber sync from a custom backend
